# Быстрый старт для дизайнера

Этот раздел поможет подготовить первый макет по BRIDGE, не читая заранее всю спецификацию. Пройдите один пример, выполните пять шагов и затем откройте [каталог примеров](../../examples/README.ru.md), если встретите спорный случай.

## Что должно получиться

Разработчик, ИИ-агент или автоматизированный инструмент открывает Figma и без дополнительных пояснений понимает:

1. какую страницу и какое состояние он видит;
2. какие корневые фреймы являются адаптивами одной страницы;
3. какие элементы соответствуют друг другу на разных ширинах;
4. куда ведут ссылки и что делают элементы управления;
5. что является контентом, декором или экспортируемым ресурсом.

## Один пример от начала до передачи

Представим два адаптива посадочной страницы и модальное окно обратной связи.

### Было: макет приходится разгадывать

```text
Desktop
  Frame 42
    Group 18
      Heading
      Text
    Button
    Button copy

Mobile final 2
  Group 91
    Title mobile
    Text
    Button

Popup
  ...
```

По такому дереву нельзя надёжно определить:

- являются ли `Desktop` и `Mobile final 2` адаптивами одной страницы;
- обозначают ли `Heading` и `Title mobile` один элемент;
- что делают две кнопки;
- связано ли окно `Popup` с одной из кнопок;
- почему на мобильном адаптиве пропала вторая кнопка.

### Стало: макет описывает себя

```text
Главная [page=home] [route=/] [bp=1440] [view=default]
  Первый экран [section=home-hero]
    hero-copy
      hero-title
      hero-subtitle
    hero-actions
      email-link [href=mailto:sales@example.com]
      contact-button [action=modal:contact-modal]

Главная [page=home] [route=/] [bp=375] [view=default]
  Первый экран [section=home-hero]
    hero-copy
      hero-title
      hero-subtitle
    hero-actions
      email-link [href=mailto:sales@example.com]
      contact-button [action=modal:contact-modal]

Окно обратной связи [modal=contact-modal]
  modal-content
    modal-title
    close-button
```

Теперь видно:

- одинаковые `[page=home]`, `[view=default]` и имена слоёв связывают два адаптива;
- `[bp=1440]` и `[bp=375]` обозначают их контрольные ширины;
- `[href=mailto:sales@example.com]` задаёт известную цель перехода;
- `[action=modal:contact-modal]` ссылается на существующее модальное окно `[modal=contact-modal]`;
- `close-button` является экземпляром элемента управления из `UI Kit`, поэтому его поведение можно получить из метаданных компонента;
- размеры, расположение, Auto Layout и стили по-прежнему читаются из Figma, а не из тегов.

> Раскладка адаптивов может отличаться. Важно сохранить один логический набор элементов и понятные отношения между ними.

## Подготовьте макет за пять шагов

### 1. Опишите корневые фреймы

Минимальная запись для каждого адаптива:

```text
Название [page=page-id] [bp=ширина] [view=default]
```

Добавьте `[route=/production-path]`, только если настоящий рабочий маршрут уже известен. Не придумывайте адрес ради заполнения тега.

### 2. Сопоставьте адаптивы

Положите рядом широкий и мобильный варианты и проверьте их слой за слоем:

- один логический элемент имеет одно имя;
- элемент остаётся у того же логического родителя;
- смысл текста и действия не меняются;
- новая обёртка появляется только по реальной структурной причине.

Размеры, направление Auto Layout, отступы, порядок внутри одного родителя и видимость могут отличаться.

### 3. Приведите имена к стабильному виду

Используйте короткие английские имена в `kebab-case`:

```text
hero-title
product-grid
contact-button
```

Не кодируйте в имени ширину или устройство:

```text
hero-title-mobile   // плохо
button-375          // плохо
```

Имя нужно не для описания внешнего вида, а для узнавания того же элемента в другом адаптиве.

### 4. Добавьте только недостающий смысл

| Figma уже хранит | BRIDGE добавляет |
| --- | --- |
| тип слоя, компонент и вариант | страницу и состояние: `[page]`, `[view]` |
| Auto Layout, интервалы и внутренние отступы | маршрут и контрольную ширину: `[route]`, `[bp]` |
| размеры, позицию и ограничения | ссылку или действие: `[href]`, `[action]` |
| заливку, обводку, эффект и маску | цель: `[modal]`, `[state]` |
| иерархию фреймов и групп | продуктовый смысл: `[section]`, `[decor]`, `[asset]` |

Не добавляйте в названия `row`, `gap=16`, `absolute` или `text`, если эти свойства уже однозначно заданы средствами Figma.

### 5. Проверьте макет без участия автора

Попросите другого человека за пять минут ответить по одному только файлу:

- Что это за страница и состояние?
- Где находятся её адаптивы?
- Что произойдёт при взаимодействии с каждым элементом?
- Где находятся цели модальных окон и состояний?
- Что нужно экспортировать, а что должно остаться редактируемым?

Если ответы приходится сообщать устно, в макете пока не хватает информации.
