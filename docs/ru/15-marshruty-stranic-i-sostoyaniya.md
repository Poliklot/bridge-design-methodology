# Маршруты и состояния страниц

BRIDGE считает страницы, рабочие маршруты, якоря и состояния данных полноценной частью контракта. Макет описывает не только секции, но и навигацию продукта.

## Идентификатор страницы и необязательный маршрут

Корневому фрейму всегда нужен стабильный `[page=...]`. `[route=...]` добавляется только тогда, когда известен настоящий рабочий адрес.

Маршрут известен:

```text
Контакты [page=contacts] [route=/contacts] [bp=1200] [view=default]
Контакты [page=contacts] [route=/contacts] [bp=320] [view=default]
```

Маршрут пока неизвестен:

```text
Контакты [page=contacts] [bp=1200] [view=default]
Контакты [page=contacts] [bp=320] [view=default]
```

Правила:

- `[page=...]` — стабильный идентификатор страницы;
- `[route=...]` — настоящий путь, а не догадка или заглушка;
- `[route-pattern=...]` — настоящий шаблон динамического маршрута;
- отсутствие маршрута в черновике является задачей на будущее, а не блокирующей ошибкой;
- все адаптивы и состояния одной страницы используют один маршрут, когда он известен;
- конкретный маршрут уникален внутри сайта, если это не объявленный шаблон.

## Синтаксис маршрута

```text
[route=/]
[route=/contacts]
[route=/catalog]
[route=/catalog/product]
[route-pattern=/catalog/:slug]
```

- `[route=...]` задаёт конкретную страницу;
- `[route-pattern=...]` задаёт реальный шаблон, например карточку товара;
- не придумывайте временный маршрут ради чеклиста;
- не добавляйте завершающий `/`, кроме корня, если продукт не требует иного;
- значения чувствительны к регистру, но рекомендуется нижний регистр и `kebab-case`.

## Состояния страницы

`[view=...]` описывает состояние страницы или данных, а не состояние отдельного компонента.

```text
Каталог [page=catalog] [route=/catalog] [bp=1200] [view=default]
Пустой каталог [page=catalog] [route=/catalog] [bp=1200] [view=empty]
Загрузка каталога [page=catalog] [route=/catalog] [bp=1200] [view=loading]
Ошибка каталога [page=catalog] [route=/catalog] [bp=1200] [view=error]
```

Типовые значения:

```text
default
empty
loading
error
filtered
not-found
unauthorized
success
```

Правила:

- у страницы должно быть `view=default`;
- динамической странице при необходимости нужны пустое состояние, загрузка и ошибка;
- все состояния используют один `[page]` и один `[route]`;
- состояние может менять содержимое, но адаптивы внутри одного состояния сохраняют его смысл;
- сравнение идентификаторов, типов, родителей и текста выполняется только между адаптивами одного `[view]`;
- каждое заявленное состояние должно иметь фрейм для каждой обязательной контрольной ширины страницы.

## Не создавайте отдельную страницу для состояния

Плохо:

```text
Каталог [page=catalog] [route=/catalog] [view=default]
Пустой каталог [page=catalog-empty] [route=/catalog-empty]
```

Хорошо:

```text
Каталог [page=catalog] [route=/catalog] [view=default]
Пустой каталог [page=catalog] [route=/catalog] [view=empty]
```

Если маршрут неизвестен, сохраните один идентификатор страницы и просто не добавляйте `[route]`.

## Секции и якоря

Адресуемая секция может иметь одновременно секционный ключ и якорь:

```text
Вопросы о контактах [section=contacts-faq] [anchor=faq]
```

`[section]` описывает повторно используемый тип секции. Название конкретного блока и его содержимое могут отличаться:

```text
Каталог [section=product-slider]
Похожие товары [section=product-slider]
Рекомендованные товары [section=product-slider]
Первый экран [section=home-hero]
```

Правила:

- не добавляйте `Секция /` в имя слоя;
- `[section]` стабилен между адаптивами одной страницы;
- одинаковый `[section]` на разных страницах допускает разное содержимое;
- якорь уникален внутри страницы;
- одна секция сохраняет один якорь между адаптивами;
- якоря определяются в макете, а не придумываются во время реализации.

## Как разрешаются ссылки

```text
Контакты [href=/contacts]
Вопросы и ответы [href=/contacts#faq]
К вопросам на этой странице [href=#faq]
Telegram [href=https://t.me/company]
```

Если адрес неизвестен, используйте `[link]`. Не используйте `[href=#]` как заглушку.

Валидатор сопоставляет:

```text
href=/contacts      → страница с [route=/contacts]
href=/contacts#faq  → страница [route=/contacts] и секция [anchor=faq]
href=#faq           → текущая страница и секция [anchor=faq]
href=https://...    → внешний адрес
```

Если маршрут целевой страницы ещё неизвестен, разрешение ссылки может оставаться задачей на будущее.

## Область уникальности

Для корневых фреймов с известным маршрутом:

```text
page + route + bp + view
```

Для черновика без маршрута:

```text
page + bp + view
```

## Что проверяет валидатор

- корневой фрейм без маршрута отмечается как задача на будущее, а не как ошибка;
- указанный маршрут соответствует формату рабочего пути;
- конкретные маршруты не повторяются;
- одна страница не меняет маршрут между адаптивами;
- каждое состояние существует на обязательных контрольных ширинах;
- динамическая страница имеет необходимые состояния;
- `[href=#]` не используется как заглушка;
- внутренний маршрут и якорь существуют;
- якорь не повторяется внутри одной страницы;
- состояние страницы не смоделировано отдельным вымышленным маршрутом.
