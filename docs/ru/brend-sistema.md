## Смысл

Знак состоит из шести последовательных модулей:

1. **B** — Breakpoints / Адаптивы
2. **R** — Roles / Роли
3. **I** — Identity / Идентичность
4. **D** — Dependencies / Зависимости
5. **G** — Geometry / Геометрия
6. **E** — Exceptions / Исключения

Модули сохраняют общий ритм и чередуют структурные блоки с открытыми пролётами. Вместе они образуют абстрактную **B**: одну идентичность из явно заданных частей, различимую в любом размере.

Знак намеренно не изображает буквальный мост. BRIDGE описывает контракт переноса, поэтому символ передаёт стабильность структуры и непрерывность, а не конкретный объект.

## Палитра

| Токен | Значение | Смысл |
| --- | --- | --- |
| BRIDGE Ink | `#1E1E1E` | Структура, техническая правда, основной логотип |
| BRIDGE Blue | `#0D99FF` | Объявленный смысл, активный путь переноса |
| White | `#FFFFFF` | Контраст и свободное пространство |

Основной знак использует только эти три цвета. Цвета состояний продукта — ошибка, предупреждение и черновик — относятся к интерфейсу и не добавляются в логотип.

## Правила

- Используйте полноцветный знак везде, где это позволяет поверхность.
- Одноцветный знак нужен для штампов, небольших технических схем и ограниченного производства.
- Охранное поле — не меньше одного внутреннего зазора между модулями.
- Минимальный размер: `16 × 16 px` для знака и `120 px` по ширине для логотипа.
- Сохраняйте порядок модулей, расстояния, геометрию углов и чередование синего с белым.
- Не добавляйте градиенты, тени, обводки, подписи или буквальную арку моста.
- Не используйте шесть модулей как случайный декор: их порядок несёт смысл.
