# Preflight-чеклист

Используй этот чеклист перед передачей макета в перенос.

## Уровни проверки

Используй чеклист в трёх режимах:

- **Quick** — self-check дизайнера перед обсуждением handoff.
- **Strict** — reviewer approval check перед началом переноса.
- **Adapter** — цель-specific проверка против возможностей реализации.

Метки автоматизации:

- **Auto** — должно проверяться валидатором.
- **Heuristic** — валидатор может подсветить подозрительный кейс.
- **Manual** — нужно подтверждение человека.

Блокеры:

- отсутствуют stable identities у важных элементов;
- отсутствуют обязательные BRIDGE-теги там, где Figma не знает смысл;
- duplicate identities внутри breakpoint/view scope;
- breakpoint-specific identity values вроде `[control=button-reviews-box-375]`;
- clickable elements без actions;
- action targets, которых не существует;
- количество элементов или вложенность меняются между адаптивами без явного исключения;
- stable decorative/asset identities отсутствуют на нужном адаптиве;
- text fixed height без overflow policy;
- динамический текст зависит от ручных переносов строк;
- hidden keyed layers как источник правды.

## Identity

- [ ] Каждый важный элемент имеет stable identity: либо имя слоя в English kebab-case, либо обязательный BRIDGE-тег.
- [ ] Имена стабильных элементов написаны на английском в kebab-case.
- [ ] BRIDGE identity values не содержат названия или ширину breakpoint; suffixes вроде `-768`, `-375`, `-mobile` или `-desktop` удалены.
- [ ] Внутри одного breakpoint/view scope нет duplicate identities.
- [ ] Один и тот же логический элемент использует одну identity на всех адаптивах.
- [ ] Один и тот же логический элемент сохраняет одного родителя на всех адаптивах.
- [ ] `[decor]` и `[asset]` трактуются как visual-intent/policy flags, а не как дополнительные identities.
- [ ] Тип текста, контентной картинки и иконки берётся из Figma, а не из ручного тега.
- [ ] Секционные компоненты лежат на странице библиотеки `Page Sections`, а обычные компоненты интерфейса — в `UI Kit`.
- [ ] Экземпляры компонентов из `Page Sections` не дублируют `[section=...]` на странице.
- [ ] Обычные секции, собранные фреймом, или неоднозначные компоненты имеют явный `[section=...]`.
- [ ] Имена секций не дублируют роль префиксом вроде `Секция /`.

## Структура Figma

- [ ] Связанные элементы собраны в осмысленные frames/groups/components, а не разложены случайными free siblings.
- [ ] Auto Layout, constraints, clip content, исходный компонент и позиционирование заданы средствами Figma, а не ручными BRIDGE-тегами.
- [ ] В названиях слоёв нет ручных тегов, дублирующих технические свойства Figma.
- [ ] Wrapper’ы имеют реальную структурную причину.
- [ ] Wrapper’ы, которые существуют только на одном адаптиве, описаны как исключения и имеют причину.

## Адаптивы

- [ ] Каждый корневой фрейм адаптива имеет `[bp=...]`.
- [ ] Ширина/название breakpoint объявлены только на root, а не повторяются в child ids.
- [ ] Важные ключи присутствуют на нужных адаптивах.
- [ ] Адаптивы сохраняют один набор логических элементов и одинаковую структуру списков/коллекций.
- [ ] В списках с постраничной навигацией количество элементов не меняется ради красивого заполнения сетки.
- [ ] Скрытые элементы всё равно сохраняют свои ключи и своих родителей на каждом нужном адаптиве.
- [ ] Элементы, которые поменялись местами, остались внутри того же родителя.
- [ ] Смысл текста не меняется скрыто между адаптивами.
- [ ] Decorative/asset root identities не исчезают между адаптивами.
- [ ] Visual-intent flags не дрейфят: `sneg [decor] [asset]` не превращается в обычный `sneg` на другом адаптиве.

## Interactions

- [ ] Каждая кнопка имеет `[action=...]`.
- [ ] Каждая ссылка имеет URL, anchor, state или цель.
- [ ] Каждый modal action ведёт на существующий modal frame.
- [ ] Каждый state action ведёт на существующий state цель.

## Height и overflow

- [ ] Текст не использует fixed height без declared overflow behavior.
- [ ] Динамический текст не зависит от forced line breaks или `<br>`, чтобы выглядеть корректно.
- [ ] Карточки не скрывают контент случайно.
- [ ] Fixed height имеет reason.
- [ ] Overflow behavior явно задан для clipped surfaces.

## Визуальный смысл

- [ ] Контентные картинки имеют стабильные имена слоёв.
- [ ] Редактируемый текст не экспортируется картинкой без причины.
- [ ] Сложные визуалы, которые надо экспортировать целиком, помечены как `[asset]`.
- [ ] Декоративные визуалы помечены как `[decor]`.
- [ ] `[decor]` layers исключены из accessibility/content semantics и могут быть `aria-hidden`.
- [ ] Текст внутри `[decor]` или root `[asset]` visuals не проверяется как product content drift, но root identity всё равно проверяется между адаптивами.
- [ ] Формы со значением вроде `[decor=snow-bg]` или `[asset=promo-poster]` используются только как fallback для плохих/default имён, а значение написано в English kebab-case.
