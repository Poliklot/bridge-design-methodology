# Высота и переполнение

Фиксированная высота — один из самых частых источников ошибок при реализации. В BRIDGE способ расчёта высоты и поведение при переполнении должны быть осознанными.

## Предпочтение по умолчанию

По возможности размер должен следовать содержимому:

- текстовый слой растёт по содержимому;
- карточка увеличивается вместе с текстом;
- секция использует внутренние отступы и минимальную высоту, а не случайную обрезку;
- одинаковая высота карточек достигается правилами сетки, а не скрытым переполнением.

## Когда допустима фиксированная высота

Фиксированная высота может быть оправдана для:

- корневого фрейма с заданной контрольной шириной;
- кнопки или элемента управления из дизайн-системы;
- иконки или изображения с постоянным соотношением сторон;
- карточек одинаковой высоты, если это продуктовое решение;
- маски, постера или экспортируемого визуала;
- прокручиваемой области с явно заданным поведением.

Укажите причину:

```text
feature-card [height=fixed] [reason=equal-card-grid]
```

## Текст и фиксированная высота

Для текстового слоя с фиксированной высотой нужно определить поведение при изменении содержимого.

Плохо:

```text
description [height=fixed]
```

Хорошо:

```text
description [height=hug]
```

Если обрезка намеренна:

```text
description [height=fixed] [overflow=truncate] [lines=3]
```

## Ручные переносы строк

Ручной перенос строки не должен заменять нормальную раскладку.

Не вставляйте принудительные переносы, `<br>` или разные версии текста для отдельных ширин только ради композиции. Текст из CMS, каталога, административной панели или файла локализации должен переноситься по ширине контейнера.

Плохо:

```text
hero-title
"Запустите зимний
бизнес быстрее"
```

если перенос нужен только как визуальный костыль.

Хорошо:

```text
hero-title [height=hug]
```

Настройте ширину блока, автоматический перенос и минимальную высоту. Используйте `[overflow=truncate] [lines=...]` только тогда, когда обрезка текста является принятым продуктовым решением.

Принудительный перенос допустим, если он является частью содержания или утверждённой композиции: например, в почтовом адресе, стихотворении, юридическом тексте или брендированном заголовке. Оформите это как исключение:

```text
campaign-title [bridge-exception=manual-line-break] [reason=brand-lockup]
```

## Правила обработки переполнения

Используйте явные значения:

```text
[overflow=visible]
[overflow=hidden]
[overflow=scroll]
[overflow=truncate]
```

Если элемент выходит за границы родителя, поместите его на тот уровень иерархии, где такое поведение действительно задумано.
