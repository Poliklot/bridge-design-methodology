# Компоненты, `UI Kit` и `Page Sections`

BRIDGE разделяет библиотечные компоненты на два уровня:

- `UI Kit` — обычные элементы интерфейса;
- `Page Sections` — готовые крупные секции страницы.

Разделение не позволяет перепутать секцию с кнопкой, карточкой или полем и избавляет от повторения `[section=...]` на каждом экземпляре секционного компонента.

## Главное правило

> Экземпляр компонента из `Page Sections` считается секцией. Экземпляр компонента из `UI Kit` секцией не считается.

Роль определяется по исходному компоненту, а не по размеру или положению экземпляра на странице.

## Два уровня библиотеки

```text
UI Kit
  button
  input
  select
  checkbox
  tabs
  accordion
  product-card
  review-card
  icon-button

Page Sections
  header
  footer
  hero
  reviews
  faq
  product-slider
  catalog-preview
  cta
```

`UI Kit` хранит строительные элементы интерфейса. `Page Sections` хранит крупные блоки, которые можно повторно использовать на разных страницах с другим содержимым.

## Как секция выглядит на странице

Экземпляры из `Page Sections` называются просто:

```text
footer
reviews
product-slider
```

Не нужно дублировать:

```text
footer [section=footer]
reviews [section=reviews]
product-slider [section=product-slider]
```

Секционный ключ берётся из исходного компонента.

## Как определяется секционный ключ

Ключ берётся из имени компонента или набора вариантов:

```text
Component set: footer
  Breakpoint=1920
  Breakpoint=1280
  Breakpoint=768
  Breakpoint=375
```

Здесь секционный ключ — `footer`. Свойство варианта `Breakpoint` в ключ не входит.

## Почему не нужно повторять `[section]`

Если ставить тег на каждый экземпляр:

- Figma и тег становятся двумя источниками одной информации;
- они могут разойтись;
- один компонент приходится размечать заново на каждой странице.

Если Figma надёжно знает исходный компонент, BRIDGE не требует повторять его смысл вручную.

## Когда `[section=...]` всё-таки нужен

Тег используется, если:

1. секция собрана обычным фреймом;
2. исходный компонент слишком общий;
3. секционный смысл намеренно переопределяется для конкретного места.

```text
Отзывы [section=reviews]
```

## Обёртки вокруг секций

Секции не обязаны быть прямыми дочерними слоями корневого фрейма:

```text
page-root
  dark-area
    reviews
    faq
```

`dark-area` отвечает за общий фон и отступы, поэтому является обёрткой. `reviews` и `faq` остаются секциями, если их исходные компоненты находятся в `Page Sections`.

## Ответственность `UI Kit`

`UI Kit` хранит компоненты и их состояния. Страница не должна быть местом, где впервые придумываются наведение, фокус, отключение, загрузка, ошибка, выбор или раскрытие.

| Семейство | Ожидаемые состояния |
| --- | --- |
| Кнопка | обычное, наведение, фокус, нажатие, отключение, загрузка |
| Ссылка | обычное, наведение, фокус, посещение, отключение |
| Поле | пустое, заполненное, фокус, ошибка, отключение, успех |
| Список выбора | закрыт, открыт, выбрано, фокус, ошибка, отключение |
| Переключатель | выключен, включён, фокус, отключение |
| Вкладки | обычная, активная, наведение, фокус, отключение |
| Аккордеон | свёрнут, раскрыт, фокус, отключение |
| Модальное окно | обычное, закрытие, фон, удержание фокуса |

Точная модель зависит от дизайн-системы. BRIDGE не заставляет перечислять состояния компонента в имени каждого экземпляра.

## Ответственность экземпляра

Экземпляр на странице описывает только смысл конкретного места:

```text
Связаться [action=modal:contact-modal]
Вопросы и ответы [href=/contacts#faq]
Почта [field=email] [name=email]
```

Не добавляйте `[component=...]`, если исходный компонент уже известен Figma.

## Что проверяет валидатор

- крупный блок не относится к `Page Sections` и не имеет `[section=...]`;
- экземпляр из `Page Sections` без причины дублирует `[section=...]`;
- компонент из `UI Kit` ошибочно используется как секция;
- один элемент на разных адаптивах связан с разными исходными компонентами;
- экземпляр отвязан от библиотеки без причины;
- обязательные состояния отсутствуют;
- состояния компонента нарисованы вручную на странице;
- переопределение экземпляра меняет запрещённую часть структуры;
- поле формы не раскрывает имя данных;
- ссылка-иконка не имеет доступного названия.

## Источник правды о компоненте

Используйте источники в таком порядке:

1. метаданные экземпляра Figma;
2. исходный компонент или набор вариантов;
3. принадлежность к `UI Kit` или `Page Sections`;
4. документация компонента;
5. BRIDGE-тег, если остальных данных недостаточно.

## Состояние страницы и состояние компонента

Не смешивайте два уровня.

Состояние страницы:

```text
Каталог [page=catalog] [route=/catalog] [bp=1200] [view=empty]
```

Состояние компонента:

```text
Button / Primary / Disabled
Input / Error
Accordion / Expanded
```

Пустой каталог является состоянием страницы. Наведение, фокус, отключение и загрузка отдельных компонентов принадлежат библиотеке.
